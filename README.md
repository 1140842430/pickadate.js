## Dependencies

To avoid running into circular depedencies, the modules are organized so they can only `require` higher or same level modules:

- `constants/`
- `language.js`
- `classes.js`
- `utils/`
- `actions/` & `reducers/`
- `addons/`
- `objects/`
- `renderers/`
